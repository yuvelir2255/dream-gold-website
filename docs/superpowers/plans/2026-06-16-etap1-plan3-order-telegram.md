# Этап 1 · План 3 — Форма заказа `/order` → Telegram

Дата: 2026-06-16. Статус: **код готов и проверен в браузере; ждём токен бота от владельца.**

## Цель
Сделать сайт рабочим инструментом для заявок: страница индивидуального заказа `/order`
с формой, которая отправляет заявку в Telegram. До этого кнопки CTA вели на несуществующую `/order`.

## Решения (зафиксировано с владельцем)
- **Без фото** на старте — текстовая форма (проще/быстрее). Фото-референс добавим отдельным шагом.
- **Одна страница с секциями** (нумерованные блоки 01→03), а не пошаговый wizard.
- Заявки приходят **только в Telegram** (Этап 1). Почта/CRM — позже.

## Что сделано
- `messages/uk.json` + `messages/en.json` — namespace `order` (тексты укр+англ, тон maison).
- `src/components/order/OrderForm.tsx` — клиентская форма:
  - Секция 01 «Ваша ідея»: textarea (обяз.) + чип «натхнення» из `?ref=` (с кнопкой «прибрати»).
  - Секция 02 «Деталі виробу»: пилюли тип/металл/проба (повторный клик снимает) + поле размера. Всё необязательно.
  - Секция 03 «Ваші контакти»: имя (обяз.) + телефон/Telegram (обяз.).
  - Валидация обязательных полей на клиенте; состояния `idle/sending/success/error`;
    экран благодарности; баннер ошибки со ссылкой-фолбеком на `t.me/IrinaBabii1982`.
  - Honeypot-поле `company` (скрыто off-screen + `aria-hidden`) — анти-спам.
- `src/app/[locale]/order/page.tsx` — кино-шапка (DarkBackdrop, как в каталоге) + форма
  в светлой секции. Форма обёрнута в `<Suspense>` (т.к. `useSearchParams`).
- `src/app/api/order/route.ts` — POST: серверная валидация (обяз. поля, лимиты длины, enum-значения),
  отброс спама по honeypot, маппинг значений в укр-подписи, отправка `sendMessage` в Telegram
  (HTML, экранирование). Токен/chat_id — из env, только на сервере.
- CTA: на карточке изделия кнопка «Хочу таке саме» теперь ведёт на `/order?ref=<название>`.
- `.env.local` — добавлены пустые `TELEGRAM_BOT_TOKEN` + `TELEGRAM_CHAT_ID` с комментарием.

## Проверено (preview, localhost:3000)
- `/order` рендерится (шапка + форма), все поля и 11 пилюль на месте.
- Валидация: пустая отправка → 3 сообщения «Будь ласка, заповніть це поле» + `aria-invalid`.
- Отправка с данными → `POST /api/order` → **503 not_configured** (токена нет) → баннер ошибки + фолбек. ✓
- Чип «натхнення» из `?ref=` отображается. ✓
- Мобильная вёрстка (375px) — ОК (чип и пилюли переносятся, поля на всю ширину).
- `tsc --noEmit` — без ошибок.
